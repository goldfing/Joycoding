# Insertion Sort

카드를 정렬하듯이 하나씩 이동하면서 알맞은 자리에 삽입하는 정렬방식이다. 

## 시간 복잡도 
> 1 + 2 + ... + (n-1) = n(n-1)/2 = O(n^2)

## 장점
- 크기가 작은 리스트의 경우 알고리즘 자체가 간단해서 다른 복잡한 정렬 방법보다 유리할 수 있다.

## 단점
- 상대적으로 시간 복잡도가 높다.
- 원소의 이동이 리스크 크기에 비례하다. 대용량 정렬에는 적합하지 않다.  

## 총평
간단하지만 비효율적이다.

## 궁금한 점
과연 효율적으로 정렬할 수 있는 리스트의 최대치는 얼마일까?
